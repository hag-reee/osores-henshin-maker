# Vercelへのデプロイ手順

このプロジェクトは **Vercel** での動作を確認済みです。以下の手順でデプロイを行ってください。

## 1. GitHubへのアップロード
このフォルダ（`github_upload_ready`）の中身を、ご自身のGitHubリポジトリにアップロードしてください。
※今回追加された `favicon.png` や `ogp.png` も含まれています。

## 2. Vercelでの設定
1. Vercelのダッシュボードにアクセスし、**"Add New Project"** をクリックします。
2. GitHubリポジトリを選択し、インポートします。
3. **Framework Preset** は自動的に `Next.js` が選択されるはずです（そのままでOK）。
4. **Build Command** や **Output Directory** もデフォルトのままでOKです。

## 3. 環境変数の設定 (重要)
**ここが最も重要です！**
デプロイを実行する前に、**Environment Variables** のセクションを開き、以下のキーを設定してください。

- **Key**: `GEMINI_API_KEY`
- **Value**: (あなたのGemini APIキーを入力してください)

※ APIキーを設定しないと、AIによる言い訳生成機能が動作しません。

## 4. デプロイ実行
設定が完了したら **"Deploy"** ボタンを押してください。
数分でビルドが完了し、公開URLが発行されます。

---
**補足:**
- ローカル環境で使っていた `.env.local` ファイルはセキュリティ保護のためGitHubにアップロードしていません。
- Vercel上で直接環境変数を設定することで、安全にキーを管理できます。
